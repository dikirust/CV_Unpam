Laporan Proyek CNN untuk Klasifikasi Sampah Otomatis

1. Pendahuluan

Latar Belakang

Klasifikasi citra merupakan salah satu aplikasi computer vision yang penting dalam kehidupan sehari-hari. Convolutional Neural Network (CNN) telah terbukti sangat efektif untuk tugas klasifikasi gambar karena kemampuannya dalam mengenali pola spasial dan fitur visual.

Tujuan

Membangun model CNN dari awal untuk mengklasifikasikan 5 jenis sampah (foodwaste, glass, metal, paper, plastic) untuk sistem pemisah sampah otomatis.

Dataset

Dataset terdiri dari gambar sampah dengan 5 kategori: foodwaste, glass, metal, paper, dan plastic. Dataset dibagi menjadi training set (8400 gambar), validation set (469 gambar), dan test set (237 gambar).

2. Metodologi

Pra-Pemrosesan

- Resize gambar ke ukuran 64x64 pixels untuk efisiensi komputasi

- Normalisasi pixel values ke range 0-1 dengan membagi nilai pixel dengan 255.0

- Konversi color space dari BGR ke RGB

Arsitektur Model

Model CNN sederhana dengan konfigurasi ringan:

  • 3 Convolutional layers dengan filter 16, 32, 32
  • MaxPooling2D setelah setiap conv layer
  • Flatten layer
  • Dense layer dengan 32 units + Dropout 0.3
  • Output layer dengan 5 units (softmax activation)

Total parameters: 80,069

Parameter Pelatihan

Optimizer: Adam (learning_rate=0.001)

Loss Function: Sparse Categorical Crossentropy

Batch Size: 16

Epochs: 8

Callbacks: EarlyStopping, ReduceLROnPlateau

3. Hasil dan Analisis

Metrik Performa

Test Accuracy: 0.6160

Test Precision: 0.5898

Test Recall: 0.6160

Test F1-Score: 0.5807

Analisis Kinerja

Analisis Overfitting

Berdasarkan training curves, model menunjukkan pelatihan yang baik. Training dan validation curves relatif seimbang.

Analisis Kesalahan

Berdasarkan confusion matrix, kelas yang paling sering salah diklasifikasikan dapat dilihat dari nilai off-diagonal yang tinggi. Hal ini mungkin disebabkan oleh kesamaan visual antar kategori sampah tertentu dalam resolusi 64x64.

4. Kesimpulan

Temuan Utama

Model CNN sederhana berhasil mencapai accuracy 61.60% pada test set. Model menunjukkan performa yang stabil.

Saran Perbaikan

  1. Data Augmentation: Menambahkan rotasi, flip, dan zoom untuk meningkatkan variasi data
  2. Regularization: Menambahkan dropout lebih banyak atau batch normalization untuk mengurangi overfitting
